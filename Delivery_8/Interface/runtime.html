Steps per block.

Each block generates a number of steps or machine level instructions that are then read by the Choo Choo Train compiler to be executed. If a block has large amount of statements, loops, branches, or list manipulations then the number of steps will be higher. The steps generated at compilation time may be different than those generated at runtime.

Total compilation steps.

Each block has its own weight on the entire program. Comparing how many steps each block has compared to others can help visualize just how much work each block is responsible for during compilation.

Total runtime steps.

Each block has its own weight on the entire program. Comparing how many steps each block has compared to others can help visualize just how much work each block is responsible for during runtime.

Variables per block.

Each block has a total amount of variables including variables that are simple data types, parameters, and lists. Some blocks contain more variables than we expect, these variables take up memory space and can sometimes saturate the available memory.

Total variables.

Each block has its own weight on the program's memory. Comparing how many variables each block has compared to others can help visualize just how much memory space each block is taking.
